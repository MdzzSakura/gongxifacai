# A股波段选股系统 — 架构设计

生成时间:2026-06-29
状态:已通过设计评审,待落地

---

## 1. 项目目标

构建一个**自上而下的 A股波段选股系统**:先用行业景气定大方向,再用个股业绩定标的,最后用量价/K线定买点,产出可执行的波段候选名单并推送。

核心投资逻辑(用户风格):
- **基本面 = 行业景气 + 个股业绩**(不只看个股业绩,还看行业近两年的发展)
- **量价/K线择时**决定买点
- **波段持有**

---

## 2. 核心理念:自上而下漏斗

```
全市场 ~5000只
  │ ① 行业景气层(慢筛,周/月更新一次)
  │    行业打分 = 指数2年趋势 + 行业整体业绩增速 + 行业新闻情感
  │    → 选出景气 Top 行业(默认前30%)
  ▼
好行业的成分股
  │ ② 个股业绩层(慢筛,季度更新)
  │    ROE≥8% & 营收增速>0 & 负债率<70% & 排除亏损/ST
  ▼
基本面池(行业达标 + 业绩达标)
  │ ③ 量价/K线择时层(快筛,每日盘后扫一遍)
  │    放量突破MA20 / 缩量回踩MA60企稳 + 量比异动
  ▼
今日信号股 ~5-20只
  │ ④ 决策评分:行业分 + 业绩分 + 技术分 加权排序
  │    (可选)LLM 复核:行业/个股/风险三角色出研判
  ▼
最终波段候选 → ⑤ 看板 + 手机推送
```

**设计精髓**:行业、业绩是「慢变量」(决定能不能买,周/季更新即可),量价是「快变量」(决定什么时候买,每天扫)。两者解耦 → 计算量小、职责清晰、可独立验证。

## 3. 技术栈(Python 全栈)

| 用途 | 选型 | 说明 |
|------|------|------|
| 语言 | Python 3.10+ | A股量化生态主流 |
| 数据源 | **AKShare** | 行业分类、行业指数、个股财务、日K、资金流、新闻,全免费 |
| 计算 | **pandas** + **TA-Lib** | 均线/MACD/量比/K线形态 |
| 缓存 | **SQLite** | 本地落库,避免重复拉数 |
| 调度 | **APScheduler** | 每日盘后 15:30 自动跑 |
| 看板 | **Flask** + **ECharts** | Web 可视化 |
| 新闻情感 | 关键词词典 / **LLM** | 词典兜底,LLM(OpenAI兼容接口)增强,可选开关 |

## 4. 分层架构与目录结构

```
gongxifacai/
├── config/
│   └── strategy.yaml          # 阈值集中配置:行业Top%、ROE下限、均线参数、权重
├── data/                      # 第1层 数据层
│   ├── fetcher.py             # AKShare 封装 + 重试/限流
│   ├── cache.py               # SQLite 读写
│   └── cleaner.py             # 复权、停牌、ST 处理
├── factors/                   # 第2层 因子层
│   ├── industry.py            # 行业:指数2年趋势、行业业绩增速
│   ├── fundamental.py         # 个股:ROE、营收增速、负债率
│   ├── technical.py           # MA/MACD/量比
│   └── patterns.py            # K线形态、支撑压力
├── strategies/                # 第3层 策略层(可插拔)
│   ├── base.py                # Strategy 抽象基类:select()
│   ├── industry_screen.py     # 行业景气筛选
│   ├── fundamental_filter.py  # 个股业绩粗筛
│   └── swing_timing.py        # 放量突破/回踩择时
├── decision/                  # 第4层 决策层
│   ├── news_sentiment.py      # 行业新闻情感(词典/LLM双模式)
│   ├── scorer.py              # 行业+业绩+技术 三合一加权评分
│   └── llm_agents.py          # 多Agent复核(可选,默认关)
├── app/                       # 第5层 展示层
│   ├── server.py              # Flask Web 看板
│   └── notifier.py            # 微信/TG/邮件推送
├── scheduler.py               # 调度:每日盘后自动跑
└── main.py                    # 串起整条流水线
```

**依赖方向**:严格单向,上层只依赖下层(展示→决策→策略→因子→数据)。每层可独立学习、单独测试、单独替换。

## 5. 各层职责与接口契约

### 第1层 数据层 (data/)
- **职责**:从 AKShare 拉数 → 清洗 → 落 SQLite 缓存,对上层屏蔽数据源细节。
- **关键接口**:
  - `fetcher.get_industry_list()` → 行业列表(申万/东财分类)
  - `fetcher.get_industry_index_hist(industry, years=2)` → 行业指数近2年日线
  - `fetcher.get_stock_financial(code)` → 个股财务(ROE、营收、负债)
  - `fetcher.get_stock_daily(code, adjust='qfq')` → 个股前复权日K
  - `fetcher.get_industry_news(industry)` → 行业相关新闻
- **约束**:带重试/限流(AKShare 接口不稳),缓存命中优先,失败不阻塞整条流水线。

### 第2层 因子层 (factors/)
- **职责**:把原始数据算成因子值,纯函数、无副作用。
- **关键接口**:
  - `industry.trend_score(index_hist)` → 行业指数2年趋势分(涨幅+均线多头)
  - `industry.earnings_growth(constituents_financials)` → 行业整体营收/利润增速
  - `fundamental.compute(financial)` → {roe, rev_growth, debt_ratio}
  - `technical.compute(daily)` → {ma20, ma60, macd, vol_ratio}
  - `patterns.detect(daily)` → 形态标记(突破/回踩/锤子线等)

### 第3层 策略层 (strategies/)
- **职责**:用因子值做筛选/择时,可插拔。每个策略实现 `Strategy.select(universe) -> list[code]`。
- **关键策略**:
  - `industry_screen`:行业打分排序,取 Top%(默认前30%)
  - `fundamental_filter`:ROE≥8% & 营收增速>0 & 负债率<70% & 排除亏损/ST
  - `swing_timing`:放量突破MA20 / 缩量回踩MA60企稳

### 第4层 决策层 (decision/)
- **职责**:汇总各维度分数,产出排序候选。
- **关键接口**:
  - `news_sentiment.score(industry)` → 行业新闻情感分(词典/LLM双模式)
  - `scorer.rank(candidates)` → 行业分+业绩分+技术分 加权排序
  - `llm_agents.review(stock)` → 多Agent研判(可选,默认关)

### 第5层 展示层 (app/)
- **职责**:Web 看板展示候选,推送到手机。
- **关键接口**:`server` 提供候选名单页面;`notifier.push(candidates)` 推送。

### 调度 (scheduler.py / main.py)
- `main.run_pipeline()` 串起整条流水线;`scheduler` 每日盘后定时触发。

## 6. 行业景气层算分细则(用户重点)

行业综合景气分 = 加权合成以下三项(权重写在 config):

1. **指数2年趋势分**:行业指数近2年涨幅 + 当前是否均线多头排列(MA20>MA60>MA120)。
2. **行业整体业绩增速分**:行业内成分股营收/净利润汇总,近2年增速是否持续为正。
3. **行业新闻情感分**:拉取行业相关新闻 →
   - 默认:关键词词典打分(利好/利空词频)
   - 增强(可选开关):LLM 解读新闻给出情感分

取综合景气分 Top%(默认前30%)的行业进入下一层。

## 7. 阈值默认值(集中在 config/strategy.yaml,可随时改)

| 项 | 默认值 |
|----|--------|
| 行业景气 Top% | 前 30% |
| 行业景气三项权重 | 趋势0.4 / 业绩0.4 / 新闻0.2 |
| 个股 ROE 下限 | ≥ 8% |
| 个股营收增速 | > 0 |
| 个股资产负债率 | < 70% |
| 排除 | 亏损、ST、退市风险 |
| 均线参数 | MA20 / MA60 |
| 决策三合一权重 | 行业0.3 / 业绩0.4 / 技术0.3 |
| LLM 复核 | 默认关闭 |

## 8. 分阶段落地路线(先窄而深)

| 阶段 | 范围 | 验证标准 |
|------|------|----------|
| **阶段1(第一版)** | 数据层 → 行业景气层 → 个股业绩层 → 输出基本面候选池(CSV/控制台表) | 能跑出一张「行业景气 + 业绩达标」的候选池表 |
| 阶段2 | 量价/K线择时层 | 在候选池上跑出 5-20 只带买点信号的票 |
| 阶段3 | 决策评分 + Web看板 + 推送 + 调度 | 每日盘后自动出看板并推送到手机 |
| 阶段4 | LLM 多Agent复核(可选) | 给候选股附 AI 研判解读 |

**第一版聚焦阶段1**:先打通「自上而下基本面漏斗」,让用户先看到全市场筛出一批行业景气且业绩达标的票,边看边学,有成就感后再往择时推进。

## 9. 关键风险与应对

- **AKShare 接口不稳/限流**:数据层统一加重试+限流+缓存,单只失败不阻塞整体。
- **行业分类口径**:统一采用一种分类(申万 或 东财),配置可切换,避免混用。
- **新闻与行业映射**:行业新闻按行业关键词检索,映射不准时降级为「无新闻分」而非报错。
- **复权/停牌/ST**:数据清洗层统一处理,因子层只拿到干净数据。
- **性能**:行业/业绩是慢变量(周/季缓存),只有量价每日全市场扫,计算量可控。

---

## 10. 测试策略

- 每层纯函数(尤其 factors/)写单元测试,用固定样本数据断言因子值。
- 数据层用缓存样本做离线测试,不依赖实时网络。
- 流水线做一次端到端冒烟测试:小范围股票池跑通全链路。
