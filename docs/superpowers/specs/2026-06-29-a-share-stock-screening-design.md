# A股选股系统 — 架构设计(净利润断层 + 板块效应 + 情绪周期)

生成时间:2026-06-29(重写)
状态:已通过设计评审,待落地
依据:用户交易体系文档(xingqiudocs/ 净利润断层理论篇·交易篇·做T·心态篇、复盘、集合竞价、看盘、冰点、量化图、复盘/情绪周期思维导图)

---

## 1. 项目目标

把用户的中短线交易体系**工具化、量化、可复盘**。核心不是泛泛选股,而是复刻这套打法:

> **情绪周期定位 → 板块/主线(赚钱效应) → 净利润断层选股 → 个股质地过滤 → 买点 → 复盘**

一句话:**先看板块涨幅,再在板块中选个股**,以业绩超预期跳空(净利润断层)为选股核心,以情绪周期与板块效应为大环境,以复盘为地基。

## 2. 用户体系提炼(设计依据)

### 2.1 核心链条
```
情绪周期定位(大环境)   →  现在能不能做、做什么风格
        ↓
板块/主线(赚钱效应)    →  "先看板块涨幅",找绕不开的核心板块
        ↓
净利润断层选股         →  业绩超预期 + 次日跳空缺口
        ↓
个股质地过滤           →  地位/辨识度 + 估值(PE/PEG/股息率) + 缺口回补风险
        ↓
买点(竞价/回调/打板)   →  集合竞价定方向,看盘做执行
        ↓
复盘                   →  盘面复盘 + 交易复盘,层层留痕
```

### 2.2 关键可量化信号

| 模块 | 文档中的可量化信号 |
|------|------|
| 日内赚钱效应 | 涨跌家数、涨停/跌停家数、炸板率、最高连板高度、两市量能、涨幅/跌幅前三板块 |
| 情绪周期阶段 | 冰点期 → 混沌期 → 主升期(前/中/后)→ 高位震荡期 → 退潮期,每段有明确特征 |
| 净利润断层 | 业绩超预期 + 次日跳空缺口(今开 > 昨高);量能三阶段(放量 → 缩量 → 稳定) |
| 缺口回补风险 | 8 种高危情形:先手资金、高估值过热、个股下降趋势、板块下降趋势、市场大跌、投机退潮、不可持续业绩、控盘庄股 |
| 个股质地 | 是否板块核心(涨幅/成交额排名)、PE/PEG/股息率 vs 行业、市值、股性(历史涨停/炸板规律) |
| 集合竞价 | 撮合规则、竞价量能、高/低于预期判定 |
| 板块效应 | 2021 年断层后涨幅 ≥100% 的票中 72% 由板块催化驱动 → 板块是第一权重 |

### 2.3 三类标的(决定打法)
1. **纯投机/题材股**:小市值多概念,按情绪走,锚点是情绪起伏。
2. **价值投机(业绩+题材)**:有业绩有叙事,机构与游资共做,锚点是板块赚钱效应。← 用户最擅长
3. **纯价值/估值回归**:低估值低关注,锚点是市盈率,做估值回归。

## 3. 数据可行性与诚实约束

**数据源:AKShare(免费)**,覆盖:业绩预告/快报、涨停跌停池、行业/概念板块行情与资金流、个股日K与估值指标、龙虎榜。

**必须明确的局限**:文档中「业绩**超预期**」的标准是「相对券商分析师一致预期」。**AKShare 无完整的分析师一致预期数据**,因此量化实现中「超预期」只能用**代理指标(proxy)**:

- 单季度净利润同比/环比大幅增长(文档"量化参与的净利润断层"恰好认可此 proxy:低位 + 小市值 + 单季度拐点)
- 叠加次日跳空缺口 + 所在板块是否走强

设计中所有涉及"超预期"之处,均以此 proxy 实现,**不假装还原券商一致预期**。集合竞价的逐笔实时数据、连板梯队等亦存在数据获取限制,在对应模块标注降级方案。

## 6. 各层职责与接口契约

### 第1层 数据层 (data/)
- 职责:AKShare 拉数 → 清洗 → SQLite 缓存,对上层屏蔽数据源细节。
- 关键接口:
  - `fetcher.get_market_breadth(date)` → 涨跌家数、涨停跌停家数、炸板率、两市成交额
  - `fetcher.get_zt_pool(date)` → 涨停池(含连板数,用于最高板/梯队)
  - `fetcher.get_sector_rank(date)` → 行业/概念板块涨幅与资金流排名
  - `fetcher.get_sector_constituents(sector)` → 板块成分股及其涨跌幅
  - `fetcher.get_yjyg(period)` / `get_yjkb(period)` → 业绩预告 / 业绩快报
  - `fetcher.get_stock_daily(code, adjust='qfq')` → 个股前复权日K
  - `fetcher.get_valuation(code)` → PE/PB/股息率
- 约束:带重试 + 限流 + 缓存命中优先;单只/单接口失败不阻塞整条流水线。

### 第2层 因子层 (factors/) — 纯函数,无副作用
- `market_emotion.compute(breadth, zt_pool)` → {涨跌家数, 涨停数, 跌停数, 炸板率, 最高板, 量能状态}
- `sector.rank(sector_data)` → 板块涨幅/资金流排序 + 板块趋势(均线多头);`sector.core_stocks(sector)` → 板块内按涨幅/成交额的核心票
- `profit_fault.detect(daily, financials)` → 是否跳空缺口(今开 > 昨高)+ 单季度净利润增速(超预期 proxy)+ 量能三阶段标记
- `valuation.compute(code)` → {pe, peg, 股息率, 行业分位}
- `gap_refill_risk.score(stock, sector, market)` → 8 种高危情形命中数 → 回补风险分

### 第3层 策略层 (strategies/)
- `emotion_cycle.classify(emotion_series)` → 当前情绪周期阶段(冰点/混沌/主升前中后/高位震荡/退潮)+ 该阶段建议风格
- `sector_leader.select(date)` → 强势板块 → 板块内核心龙头(地位/辨识度 proxy:涨幅+成交额+带动性)
- `profit_fault_scan.select(period)` → 扫描出"业绩大增 + 跳空缺口"的断层候选

### 第4层 决策层 (decision/)
- `scorer.rank(candidates)` → 综合分 = 情绪周期分 + 板块强度分 + 断层强度分 + 质地分 − 缺口回补风险分
- `playbook.advise(stock, cycle)` → 按情绪周期给买点建议:主升期→竞价/打板,震荡期→回调低吸,冰点→绕不开的核心低吸

### 第5层 复盘层 (review/)
- `daily_board.build(date)` → 盘面复盘对象:日内赚钱效应 + 涨跌幅前三板块及其核心逻辑 + 盘面小结 + 明日关注板块
- `journal.record(trade)` → 交易留痕(标的/时间/仓位/逻辑/盈亏/评分),供周/月复盘

### 第6层 展示层 (app/)
- `server` → Flask 每日复盘面板(情绪温度计 + 板块榜 + 断层候选);`notifier.push()` → 推送当日情绪与候选。

### 调度 (scheduler.py / main.py)
- `main.run_daily()` 串起盘后流水线;`scheduler` 每日盘后定时触发。

## 7. 情绪周期判定细则(strategies/emotion_cycle.py)

依据"情绪周期"思维导图,用市场情绪因子序列判定阶段:

| 阶段 | 量化特征(proxy) | 建议风格 |
|------|------|------|
| 冰点期 | 下跌家数近期最多、最高板近期最低、领跌方向亏钱效应减缓 | 绕不开的核心低吸 / 空仓 |
| 混沌期 | 无明显主线、最高板低、轮动快、冲高回落多 | 有辨识度的核心、低仓 / 空仓 |
| 主升前期 | 出现主线、龙头打开高度、有带动性 | 买龙头 / 跟风套利 |
| 主升中期 | 龙头不断新高、梯队形成、中军启动、新分支出现 | 买龙头 / 跟风 |
| 主升后期 | 一致加速、龙头一字、逻辑扩散远 | 减仓 / 等待 |
| 高位震荡 | 龙头断板、中军冲高回落、跟风大跌、补涨频出 | 龙头做T / 补涨 / 新分支 |
| 退潮期 | 龙头破位、板块无差别杀 | 不听不看不买 / 切新板块 |

## 8. 净利润断层扫描细则(factors/profit_fault.py + strategies/profit_fault_scan.py)

断层候选 = 同时满足:
1. **业绩信号(超预期 proxy)**:业绩预告/快报显示单季度净利润同比或环比大幅增长(阈值在 config)。
2. **跳空缺口**:公告后下一交易日 今开 > 昨高(允许"长阴次日跳空无缺口"的特例)。
3. **量能三阶段**(辅助确认):放量 → 缩量 → 稳定的形态标记。

随后由 `gap_refill_risk` 过滤掉 8 种高危回补情形,由 `valuation` 标注估值是否低于行业(PEG 优先,规避"低PE零增长"价值陷阱)。

---

## 9. 缺口回补风险(factors/gap_refill_risk.py)

对每个断层候选检测 8 种高危情形,命中越多风险越高:
1. 先手资金(出业绩前已大幅上涨) 2. 高估值情绪过热(板块末期、估值分位高)
3. 个股处于下降趋势(60日线之下) 4. 板块处于下降趋势
5. 市场大跌环境 6. 投机退潮(纯题材、基本面差)
7. 不可持续业绩(一次性收益:出售资产/补贴/减值转回/大额分红)
8. 控盘庄股(换手极低、走势独立、股东集中度高)

强势断层的反向特征(加分):缺口不回补、板块/个股处趋势中、估值较低、无先手资金、与市场情绪共振。

## 10. 阈值默认值(config/strategy.yaml,可随时改)

| 项 | 默认值 |
|----|--------|
| 情绪冰点:下跌家数阈值 | 近 60 日分位 ≥90% 且最高板近期最低 |
| 市场过热:上涨家数阈值 | ≥4500 家视为日内情绪高点 |
| 断层:单季度净利润同比增速 | ≥50%(proxy 超预期) |
| 断层:跳空缺口幅度 | 今开 > 昨高(>0%) |
| 估值:PEG 上限 | ≤1.5(优先低于行业平均) |
| 缺口回补风险:高危命中阈值 | ≥2 项标红预警 |
| 综合权重 | 情绪0.2 / 板块0.3 / 断层0.3 / 质地0.2,回补风险作减分 |

## 11. 分阶段落地路线(先窄而深)

| 阶段 | 范围 | 验证标准 |
|------|------|----------|
| **阶段1(第一版)** | 数据层 + 市场情绪温度计 + 板块涨幅榜 + 净利润断层扫描器 → 输出「每日盘后复盘面板」(控制台/CSV) | 盘后能一眼看到:今日情绪指标、涨幅前列板块及成分、当期出断层的候选票 |
| 阶段2 | 情绪周期自动判定 + 个股质地评分 + 缺口回补风险过滤 | 候选票带情绪阶段标签、质地分、回补风险预警 |
| 阶段3 | 决策综合打分 + 买点建议 playbook + Flask 看板 + 推送 + 调度 | 每日盘后自动出面板并推送 |
| 阶段4 | 交易复盘留痕 journal + 周/月复盘汇总 | 可记录每笔交易并按情绪/带动性/主线/辨识度/计划内评分 |

**第一版聚焦阶段1**:复刻"先看板块涨幅"的盘后复盘地基,让用户每天盘后一眼掌握情绪、板块、断层候选,边用边学,再逐阶段加深。

## 12. 关键风险与应对

- **"超预期"无券商一致预期数据**:用单季度增速 proxy,文档已认可此 proxy(量化断层),并在面板标注"proxy口径"。
- **AKShare 接口不稳/限流**:数据层统一重试 + 限流 + 缓存,单接口失败降级不阻塞。
- **集合竞价/连板梯队实时数据受限**:阶段1不依赖实时竞价;连板高度用涨停池连板数近似,数据缺失时降级标注。
- **板块分类口径**:行业用一种口径(申万或东财),概念板块单列,配置可切换,避免混用。
- **过拟合历史**:情绪周期阈值来自经验,设为可配置参数,避免写死;面板只做"呈现+提示",不做自动下单。

## 13. 测试策略

- 因子层(market_emotion / profit_fault / gap_refill_risk 等)纯函数,用固定样本数据写单元测试断言。
- 数据层用缓存样本离线测试,不依赖实时网络。
- 端到端冒烟测试:取某一历史交易日,跑通"情绪 + 板块 + 断层扫描 → 面板"全链路。
- 关键校验:用文档中的真实案例(如山西焦化、天鹅股份、盛和资源)回测断层检测与回补风险判定是否合理。
